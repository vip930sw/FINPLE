# FINPLE Step 90. 베타 운영 기록 / 개선 로그 양식

## 1. 목적

이 문서는 FINPLE 베타 운영 중 발생한 이슈, 사용자 피드백, 개선 작업, 배포 내역을 누적 기록하기 위한 운영 로그입니다.

기능이 늘어날수록 무엇을 언제 왜 바꿨는지 추적하기 어려워지므로, 베타 단계부터 간단한 기록 체계를 유지합니다.

---

## 2. 운영 로그 기본 원칙

1. 모든 피드백을 바로 개발하지 않습니다.
2. 운영 중 발견된 문제는 우선순위를 나눕니다.
3. 실제 사용자에게 영향을 준 문제는 반드시 기록합니다.
4. 수정한 내용은 배포일과 함께 기록합니다.
5. 보류한 내용은 보류 사유를 적습니다.
6. 투자 유의 문구, 개인정보, 결제 관련 이슈는 별도 표시합니다.

## 8. 개선 backlog

### P0. 즉시 수정

- 사이트 접속 불가
- 서버 저장/불러오기 실패
- 관리자 문의 조회 실패
- 개인정보 또는 토큰 노출 가능성
- 투자 추천으로 오해될 수 있는 문구

### P1. 빠른 개선

- 모바일 주요 화면 깨짐
- 요금제 문구 혼란
- 포트폴리오 제한 정책 오류
- 차트 잘림
- 문의사항 처리 흐름 불편

### P2. 추후 개선

- 실제 이메일 로그인
- 실제 결제/구독
- Pro 플랜 기능
- 모바일 전용 시뮬레이터 입력 UI
- 리밸런싱 추천 기능
- 사용자별 포트폴리오 히스토리
- PDF 리포트 디자인 고도화

## 10. 운영 중 보류 기준

아래 항목은 베타 초기에는 바로 개발하지 않고, 사용자 반응을 본 뒤 결정합니다.

| 항목 | 보류 사유 | 재검토 기준 |
|---|---|---|
| 실제 회원가입 | 인증/보안/개인정보 처리 복잡도 증가 | 반복 사용자가 생길 때 |
| 실제 결제 | PG/환불/약관/세무 처리 필요 | 유료 의향 피드백 확인 후 |
| Pro 플랜 | 고급 기능 정의 필요 | Personal 수요 확인 후 |
| 모바일 입력 UI | 개발 범위 큼 | 모바일 사용자 비중 확인 후 |
| 알림 기능 | 이메일/푸시 인프라 필요 | 문의량 증가 후 |

---

## 11. 베타 운영 시 주의 문구

베타 기간 동안 사용자에게 아래 메시지를 반복적으로 안내합니다.

FINPLE은 현재 체험 운영 단계의 투자 분석 도구입니다. 분석 결과는 투자 판단을 돕기 위한 참고자료이며, 특정 금융상품의 매수·매도 추천이나 수익 보장을 의미하지 않습니다. 최종 투자 판단과 책임은 이용자 본인에게 있습니다.
