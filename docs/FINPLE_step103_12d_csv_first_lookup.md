# FINPLE Step 103-12D — CSV-first Lookup Policy

## 작업 개요

시뮬레이터의 조회 흐름에서 원격 API보다 로컬 CSV 후보 데이터를 먼저 사용하도록 정리했습니다.

## 변경 사항

```text
- 티커 조회 시 로컬 CSV 후보를 먼저 검색
- CSV 후보가 있으면 CAGR / BETA / MDD / dividendYield를 먼저 적용
- 미국 자산은 가격 조회 API 결과에 CSV 지표를 병합
- 한국 자산은 현재가 API 연결 전까지 CSV 지표만 적용하고 현재가는 수동 확인 유지
- 추천 프리셋 포트폴리오도 CSV 후보 지표로 초기화
```

## 적용 범위

```text
1. MBTI 또는 프리셋에서 시뮬레이터로 연결되는 기본 자산
2. 스크리너에서 시뮬레이터로 추가되는 후보 자산
3. 시뮬레이터에서 개별 조회 버튼을 누르는 자산
4. 시뮬레이터에서 전체 조회 버튼을 누르는 자산
```

## 배당률 정책

```text
- CSV에 dividendYield 값이 있으면 카드와 시뮬레이터 표에 표시
- CSV에 dividendYield 값이 없으면 “확인 중”으로 표시
- 실제 0.00인 경우에는 0.00으로 표시
```

## 주의 사항

```text
- CSV 지표는 가격 조회와 별개로 먼저 적용됩니다.
- 한국 자산은 현재가 API가 아직 없으므로 가격은 계속 수동 확인 대상입니다.
- 한국 배당/분배금은 별도 보강 대상입니다.
```
