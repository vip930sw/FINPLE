# FINPLE Step 86. 공개 전 보류 기능 / 개선 backlog

작성 목적: 베타 공개 전 반드시 고쳐야 하는 항목과, 공개 이후 개선해도 되는 항목을 분리합니다.  
작성 기준일: 2026-05-09

---

## 1. 현재 공개 가능 상태 요약

FINPLE은 현재 베타 공개 기준으로 핵심 흐름이 동작합니다.

- 도메인 연결: `https://finple.co.kr`
- 관리자 전용 주소: `https://finple.co.kr/admin`
- 일반 사용자 로그인: 체험 계정 기반
- 포트폴리오 시뮬레이터: 사용 가능
- Free 플랜: 포트폴리오 1개 제한 적용
- 서버 저장 / 불러오기: 기본 동작 확인
- 문의사항 작성 / 관리자 조회 / 상태 변경: 확인 완료
- 개인정보처리방침 / 이용약관 / 투자 유의사항: 초안 페이지 추가 완료
- 모바일 핵심 화면: 베타 기준 확인 완료

---

## 2. 공개 전 반드시 유지해야 할 기준

아래 항목은 공개 전 최종 점검 때 반드시 정상이어야 합니다.

| 구분 | 기준 | 상태 |
|---|---|---|
| 메인 도메인 | `finple.co.kr` 직접 접속 가능 | 확인 필요 |
| 관리자 주소 | `/admin`에서만 관리자 로그인 노출 | 확인 필요 |
| 관리자 토큰 | 토큰 없이는 문의 관리자 패널 미노출 | 확인 필요 |
| Free 제한 | 포트폴리오 1개만 유지 | 확인 필요 |
| 서버 저장 | 저장 후 새로고침해도 데이터 유지 | 확인 필요 |
| 문의사항 | 신규 문의 등록 및 관리자 조회 가능 | 확인 필요 |
| 법적 문서 | 개인정보처리방침 / 이용약관 / 투자 유의사항 접근 가능 | 확인 필요 |
| 투자 고지 | 수익 보장 아님, 투자 책임 본인 고지 노출 | 확인 필요 |

## 3. 공개 전 HOLD 항목

아래 항목은 아직 정식 기능으로 제공하지 않습니다.  
화면에는 “준비 중”, “체험 운영”, “제한” 등으로 표현합니다.

### 3-1. 실제 회원가입 / 이메일 로그인

현재는 체험 계정 기반입니다.

- 실제 이메일 회원가입 미연동
- 비밀번호 인증 미연동
- 소셜 로그인 미연동
- 비밀번호 찾기 미연동

공개 시 표현:

> 현재는 체험 계정으로 포트폴리오 저장과 분석 기능을 이용할 수 있습니다.

---

### 3-2. 실제 결제 / 구독

현재 요금제는 선택 상태만 저장합니다.

- 카드 결제 미연동
- 구독 갱신 미연동
- 영수증 / 세금계산서 미연동
- 결제 실패 / 환불 처리 미연동

공개 시 표현:

> 실제 결제 기능은 준비 중입니다. 현재는 플랜 기준과 제한값을 미리 확인할 수 있습니다.

---

### 3-3. Pro 플랜

현재 Pro 플랜은 준비 중입니다.

- 고급 백테스트 미구현
- 업무용 리포트 미구현
- 팀 기능 미구현
- 별도 관리자 권한 미구현

공개 시 표현:

> Pro 플랜은 준비 중입니다.

---

### 3-4. 포트폴리오 삭제 / 복구 고도화

현재 삭제 기능은 일부 제한적입니다.

- 서버 DB의 완전 삭제 기능 미제공
- 삭제 이력 / 복구 기능 미제공
- 사용자별 삭제 권한 정책 미정

베타에서는 “현재 포트폴리오 삭제” 정도만 유지하고, 서버 데이터 정리는 추후 관리자 기능으로 분리합니다.

---

### 3-5. 모바일 시뮬레이터 고도화

현재 모바일 핵심 화면은 확인됐지만, 복잡한 표/차트는 PC 사용을 권장합니다.

보류 항목:

- 모바일 표 입력 UX 고도화
- 포트폴리오 비교 차트 모바일 최적화
- 상세분석 표 고정 헤더 완성도 개선
- 모바일 리포트/PDF 보기 최적화

공개 시 표현:

> 복잡한 포트폴리오 입력과 상세 분석은 PC 환경에서 더 편하게 이용할 수 있습니다.

---

### 3-6. API 조회량 / 외부 데이터 안정화

현재 외부 데이터 조회는 Alpha Vantage와 캐시 기준으로 동작합니다.

보류 항목:

- 실시간 시세 정확도 보장 불가
- 일부 지표 수동 입력 유지
- API 한도 초과 시 캐시값 또는 기존값 유지
- 데이터 공급자 변경 가능성 존재

공개 시 표현:

> 외부 API 데이터는 지연되거나 누락될 수 있으며, 중요한 투자 판단 전 공식 자료를 별도로 확인해야 합니다.

## 4. 공개 전 FIX 항목

현재 공개 전 반드시 수정해야 하는 차단 이슈는 없습니다.  
다만 최종 공개 직전 아래 항목을 다시 확인합니다.

| 항목 | 확인 방법 | 상태 |
|---|---|---|
| Free 포트폴리오 1개 제한 | 새로고침 / 서버 불러오기 후 1개 유지 | 재확인 |
| 서버 저장 / 불러오기 | 수량 변경 → 저장 → 새로고침 → 불러오기 | 재확인 |
| 문의 등록 | 신규 문의 작성 후 관리자 조회 | 재확인 |
| 관리자 로그아웃 | 관리자 모드 해제 후 패널 숨김 | 재확인 |
| 모바일 홈 | iPhone SE / Galaxy S8+ 기준 주요 화면 확인 | 재확인 |

## 5. 공개 이후 개선 backlog

### 우선순위 A

- 실제 회원가입 / 로그인 연결
- 사용자별 서버 저장 권한 분리
- 문의사항 삭제 / 숨김 / 검색 기능
- 결제 기능 연동 전 상세 정책 정리
- 개인정보처리방침 / 이용약관 법무 검토

### 우선순위 B

- 모바일 시뮬레이터 입력 UX 개선
- PDF 리포트 디자인 개선
- 포트폴리오 비교 차트 옵션 추가
- 자산별 상세 리포트 추가
- 리밸런싱 제안 기능 추가

### 우선순위 C

- 커뮤니티 / 공유 링크
- 포트폴리오 템플릿 마켓
- 경제지표 알림
- 이메일 리포트 발송
- 다크모드

> FINPLE은 현재 베타 서비스로 운영됩니다.

> 외부 데이터는 지연되거나 누락될 수 있으므로, 실제 투자 전 공식 자료와 증권사 정보를 반드시 확인해 주세요.
