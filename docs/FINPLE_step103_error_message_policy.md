# FINPLE Step 103. 운영용 오류 안내 문구 정책

## 1. 목적

베타 운영 중 사용자가 오류를 만났을 때, 기술적인 원인보다 **지금 무엇을 하면 되는지**를 먼저 알 수 있도록 안내 문구를 정리합니다.

## 2. 주요 오류 상황

| 상황 | 사용자 안내 방향 | 우선순위 |
| --- | --- | --- |
| Render Free 인스턴스 첫 요청 지연 | 첫 요청은 30~60초 지연될 수 있음을 안내 | P2 |
| 서버 저장 실패 | 네트워크/서버 대기 가능성을 안내하고 재시도 유도 | P1 |
| 서버 불러오기 실패 | 현재 데이터가 교체되지 않았음을 전제로 재시도 유도 | P1 |
| 인증/권한 오류 | 체험 계정 또는 관리자 토큰 확인 안내 | P1 |
| 외부 API 조회 제한 | 기존 캐시/수동 입력값 유지 안내 | P2 |
| 접속 불가 | 즉시 확인 필요 | P0 |

## 3. 적용된 문구 기준

### 서버 저장 / 불러오기 진행 중

> 서버에 포트폴리오를 저장하는 중입니다. 첫 요청은 잠시 걸릴 수 있습니다...

### 네트워크 또는 서버 대기 상태

> 서버가 잠시 대기 상태이거나 네트워크 연결이 불안정할 수 있습니다. 30~60초 후 다시 시도해 주세요.

### 인증/권한 문제

> 체험 계정 연결 상태를 다시 확인해 주세요.

## 4. 운영 중 확인 기준

- 같은 사용자가 같은 오류를 2회 이상 제보하면 P1로 분류합니다.
- 여러 사용자가 동시에 접속/저장 불가를 제보하면 P0로 분류합니다.
- Render 첫 요청 지연은 Free 인스턴스 특성이므로 단독으로는 P0가 아닙니다.
- 저장 실패 후 새로고침으로 데이터가 사라지는지 반드시 확인합니다.

## 5. Step 103 완료 기준

- 서버 저장 실패 메시지가 기술 용어만 노출되지 않습니다.
- 서버 불러오기 실패 메시지가 사용자의 다음 행동을 안내합니다.
- 서버 저장/불러오기 설명에서 Supabase 등 내부 기술명이 과도하게 노출되지 않습니다.
- 베타 운영 중 문의가 들어왔을 때 P0/P1/P2 분류가 가능합니다.
