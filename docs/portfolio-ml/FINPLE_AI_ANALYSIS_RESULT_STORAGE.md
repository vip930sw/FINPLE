# FINPLE AI Analysis Result Storage

작성일: 2026-06-26
작업 단계: Step 113-4F

## 요약

STEP 4 AI 분석 결과는 생성 비용과 응답 시간이 모두 있기 때문에, 사용자가 페이지를 벗어나거나 운영 배포가 갱신되어도 같은 브라우저에서 최근 분석 결과를 다시 확인할 수 있어야 한다. 이번 단계에서는 서버 DB 변경 없이 프론트엔드의 사용자/게스트 스코프 localStorage에 포트폴리오별 최신 AI 분석 결과 1건을 저장한다.

## 구현 범위

- 저장 키: `finple-ai-analysis-cache-v1:{portfolioId}`
- 저장 스코프: 기존 포트폴리오 저장소와 동일하게 로그인 사용자 또는 게스트로 분리
- 저장 시점: AI 분석 API 응답이 validator를 통과하고 화면에 표시되는 시점
- 복원 시점: STEP 4 패널 진입, 새로고침, 재배포 후 브라우저 재방문
- 입력 변경 감지: 저장된 `inputSignature`와 현재 입력 서명이 다르면 기존 분석을 보여주되 `입력 변경됨` 상태로 표시

## 주의점

- 이 저장은 같은 브라우저의 편의 저장이며, 계정 간 서버 동기화가 아니다.
- 브라우저 데이터 삭제, 시크릿 모드 종료, 다른 기기 접속 시에는 복원되지 않는다.
- 실패한 AI 응답은 저장하지 않는다.
- 기존 저장 결과가 손상된 JSON이면 해당 캐시만 제거하고 기본 준비 상태로 돌아간다.
- 분석 결과는 투자 조언이 아니며, 저장된 결과도 최신 시장 데이터나 사용자의 새 입력을 자동 반영하지 않는다.
