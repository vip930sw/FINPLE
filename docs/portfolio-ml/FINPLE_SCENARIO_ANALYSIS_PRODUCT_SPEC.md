# FINPLE Scenario Analysis Product Specification

작성일: 2026-06-26  
대상 저장소: `vip930sw/FINPLE`  
단계: Step 114 기획  
상태: 구현 전 제품·통계 기준 문서

## 1. 문서 목적

이 문서는 FINPLE 포트폴리오 시뮬레이터에 `시나리오 분석`을 추가할 때 필요한 제품 정의, 통계 해석, 지표 관계, 차트 구조, 시장 벤치마크 기준, 사용자 고지 원칙을 확정한다.

시나리오 분석의 목적은 미래 수익률을 맞히는 것이 아니다.

```text
기존 CAGR 기반 기준선
+ 실제 변동을 반영한 수익률 경로
+ 하락확률과 하락규모
+ MDD와 회복기간
+ 시장 벤치마크 비교
= 포트폴리오 위험을 확률과 경로로 설명하는 분석
```

시나리오 분석은 기존 STEP 3 상세분석과 STEP 4 AI 분석을 대체하지 않는다.

```text
STEP 3 상세분석
= 현재 입력값과 지표를 정해진 수식과 규칙으로 설명

시나리오 분석
= 변동성, 상관관계, 하락경로, 시장충격을 반영해 분포와 경로를 계산

STEP 4 AI 분석
= 검증된 계산 결과를 사용자 문장으로 해석
```

## 2. 핵심 결론

### 2.1 현재 차트의 성격

현재 `실질 평가금액 비교` 차트는 자산 또는 포트폴리오 CAGR을 매 기간 동일하게 적용하는 기준 전망에 가깝다.

```text
고정 CAGR
→ 매년 동일한 기대수익률
→ 복리로 완만하게 우상향
→ 중간 하락·회복 경로 없음
```

현재 차트는 제거하지 않는다. 명칭을 명확히 한다.

```text
권장 명칭: 기준 CAGR 전망
설명: 입력된 CAGR이 기간 내 일정하게 유지된다고 가정한 장기 기준선
```

### 2.2 시나리오 반영 후 차트

기간별 수익률을 다르게 적용하면 상승, 하락, 횡보, 회복 구간이 생긴다.

```text
예시 연간 수익률
+12%, -8%, +18%, -15%, +11%, +7%
```

단, 월 투자금이 계속 유입되는 평가금액 차트에서는 시장이 하락해도 추가 납입금 때문에 총 평가금액이 상승할 수 있다.

따라서 FINPLE은 아래 차트를 분리한다.

| 차트 | 목적 | 월 납입금 | MDD 계산 기준 |
|---|---|---:|---:|
| 실질 평가금액 시나리오 | 실제 납입을 포함한 자산가치 범위 | 반영 | 사용하지 않음 |
| 투자위험 NAV 경로 | 순수한 시장 수익률 경로 | 제외 | 사용 |
| Drawdown 차트 | 고점 대비 하락률 | 제외 | 사용 |

### 2.3 BETA 반영 원칙

BETA는 하락확률이나 MDD에 단순히 곱하지 않는다.

```text
잘못된 방식
하락확률 × BETA × MDD
```

BETA는 시장 충격이 자산 또는 포트폴리오 수익률에 전달되는 정도를 나타내는 내부 변수다.

```text
시장 시나리오 생성
→ BETA로 시장충격 전달
→ 자산 고유충격 추가
→ 포트폴리오 경로 생성
→ 하락확률·손실규모·MDD 산출
```

단, 과거 실제 수익률을 재표본추출하는 방식에서는 BETA가 이미 가격경로 안에 내재돼 있으므로 다시 적용하지 않는다.

| 시나리오 방식 | BETA 처리 |
|---|---|
| 실제 과거 자산수익률 재사용 | 추가 적용 금지 |
| 공동 Block Bootstrap | 추가 적용 금지 |
| 시장요인 기반 Factor Model | 명시 적용 |
| 사용자가 시장 -20%를 지정하는 스트레스 테스트 | 명시 적용 |

## 3. 제품 범위

## 3.1 사용자 기능명

사용자에게는 다음 명칭을 사용한다.

```text
시나리오 분석
하락위험 시나리오
시장 스트레스 테스트
시장 기준 비교
```

내부 개발 용어는 다음과 같이 구분한다.

| 내부 용어 | 사용자 표시 |
|---|---|
| deterministic projection | 기준 CAGR 전망 |
| probabilistic simulation | 확률 시나리오 |
| block bootstrap | 과거 변동 기반 시나리오 |
| factor shock | 시장 스트레스 테스트 |
| loss probability | 손실확률 |
| expected shortfall magnitude | 손실 발생 시 평균 하락률 |
| drawdown distribution | 최대낙폭 분포 |
| benchmark composite | 가중 시장 기준 |

## 3.2 초기 제공 모드

```text
[기준 전망] [확률 시나리오] [스트레스 테스트]
```

### 기준 전망

현재 CAGR 기반 계산을 유지한다.

- 입력된 CAGR을 기간 내 일정하게 적용
- 초기 투자금과 월 투자금 반영
- 물가상승률 반영
- 배당 기준은 기존 FINPLE 정책 유지
- 단일 기준선 제공

### 확률 시나리오

과거 월간 수익률과 자산 간 공동 움직임을 이용해 복수 경로를 생성한다.

- 1년, 5년, 10년 분석
- 10,000회 시뮬레이션 기본
- P10, P50, P90 결과 제공
- 손실확률, MDD, 회복기간 제공
- 단일 랜덤 경로를 미래 예측값처럼 강조하지 않음

### 스트레스 테스트

사용자가 시장 충격을 선택한다.

```text
시장 -10%
시장 -20%
시장 -30%
금리 +1%p
원·달러 +10%
원·달러 -10%
```

MVP에서는 주식시장 충격과 BETA만 우선 지원할 수 있다. 금리·환율·신용스프레드 요인은 데이터 확보 후 단계적으로 추가한다.

## 4. 핵심 통계 정의

## 4.1 12개월 손실확률

월말 가격 기준 Rolling 12개월 수익률을 계산한다.

```text
R_12m(t) = P(t) / P(t-12) - 1
```

```text
12개월 손실확률
= Rolling 12개월 수익률이 0 미만인 구간 수
  / 전체 유효 Rolling 구간 수
```

사용자 표시:

```text
12개월 손실확률 26%
```

화면에는 반드시 데이터 기간과 유효 표본 수를 함께 표시한다.

## 4.2 손실 문턱별 확률

```text
P(12개월 수익률 < 0%)
P(12개월 수익률 ≤ -10%)
P(12개월 수익률 ≤ -20%)
P(12개월 수익률 ≤ -30%)
```

## 4.3 손실 발생 시 평균 하락률

손실이 발생한 Rolling 구간만 대상으로 평균과 중앙값을 계산한다.

```text
손실기간 평균 수익률
손실기간 중앙값
최악의 12개월
```

## 4.4 MDD

NAV 기준으로 고점 대비 하락률을 계산한다.

```text
Drawdown(t) = NAV(t) / runningMaxNAV(t) - 1
MDD = min(Drawdown)
```

추가 납입금은 NAV 위험경로와 MDD 계산에서 제외한다.

## 4.5 회복기간

고점에서 하락이 시작된 이후 동일한 고점을 다시 회복할 때까지의 개월 수를 측정한다.

```text
회복기간 중앙값
하위 10% 장기 회복기간
미회복 시나리오 비율
```

## 4.6 시나리오별 결과

각 생성 경로마다 다음을 기록한다.

```text
만기 평가금액
만기 수익률
손실연도 수
최악의 연간수익률
MDD
MDD 발생시점
회복기간
만기 손실 여부
```

집계 결과는 다음처럼 표시한다.

```text
5년 내 20% 이상 MDD 확률
5년 내 30% 이상 MDD 확률
10년 누적수익률 음수 확률
P10 / P50 / P90 만기 실질평가금액
```

## 5. 포트폴리오 계산 원칙

## 5.1 자산별 확률의 단순 가중평균 금지

자산 A 손실확률과 자산 B 손실확률을 비중가중해 포트폴리오 손실확률로 사용하지 않는다.

```text
잘못된 예
A 손실확률 30% × 50%
+ B 손실확률 20% × 50%
= 포트폴리오 손실확률 25%
```

자산 간 상관관계와 동시 하락 여부가 반영되지 않기 때문이다.

## 5.2 동일 날짜 수익률 정렬

포트폴리오 수익률은 동일 날짜의 자산 수익률을 정렬한 후 계산한다.

```text
Rp(t) = Σ wi(t-1) × Ri(t)
```

리밸런싱 주기는 별도 설정값으로 관리한다.

MVP 권장값:

```text
기본: 연 1회
선택: 리밸런싱 없음
추후: 분기 / 반기
```

## 5.3 공동 Block Bootstrap

여러 자산의 같은 기간 블록을 함께 추출한다.

```text
QQQ 2018.01~2018.12
SCHD 2018.01~2018.12
TLT 2018.01~2018.12
GLD 2018.01~2018.12
```

각 자산을 따로 무작위 추출하면 실제 상관관계가 훼손되므로 금지한다.

MVP 권장 블록 길이:

```text
6개월
12개월
```

기본값은 12개월로 두고, 검증 과정에서 6개월 결과와 비교한다.

## 6. BETA 정책

## 6.1 정의

```text
Ri(t) = alpha + beta × Rm(t) + epsilon(t)
```

- `Rm`: 시장 수익률
- `beta`: 시장 민감도
- `epsilon`: 자산 고유충격

## 6.2 시장 비교선의 BETA

시장지수를 자기 자신과 비교할 때 BETA는 정의상 1이다.

```text
S&P 500 vs S&P 500 = 1.00
KOSPI 200 vs KOSPI 200 = 1.00
```

## 6.3 포트폴리오 BETA

초기 화면용 근사값은 비중가중평균을 사용할 수 있다.

```text
betaPortfolio = Σ wi × betaAsset(i)
```

정교한 시나리오 엔진에서는 실제 포트폴리오 수익률을 생성한 후 벤치마크 수익률에 회귀해 포트폴리오 BETA를 재계산한다.

## 6.4 하락장 BETA

일반 BETA는 상승장과 하락장을 함께 포함한다. 시나리오 고도화 단계에서는 다음을 분리한다.

```text
전체 BETA
상승장 BETA
하락장 BETA
급락장 BETA
```

사용자 표시 권장명:

```text
시장 민감도
하락장 민감도
```

## 6.5 자산군별 한계

단일 주식시장 BETA로 모든 자산을 설명하지 않는다.

| 자산군 | 추가로 필요한 요인 |
|---|---|
| 장기채 | 금리·듀레이션 |
| 회사채 | 금리·신용스프레드 |
| 금 | 실질금리·달러·인플레이션 |
| 한국주식 | 국내시장·환율 |
| 해외ETF | 현지시장·환율 |
| 리츠 | 주식시장·금리 |
| 원자재 | 달러·수급·물가 |

MVP에서는 이 한계를 명시하고, 다요인 모델은 후속 단계로 분리한다.

## 7. 시장 벤치마크 정책

## 7.1 기본 벤치마크

| 포트폴리오 구성 | 기본 비교시장 |
|---|---|
| 미국 자산 중심 | S&P 500 Total Return |
| 한국 자산 중심 | KOSPI 200 Total Return |
| 한국·미국 혼합 | 가중 합성 벤치마크 |

ETF보다 지수 원본을 우선한다.

```text
미국: S&P 500 TR → S&P 500 Price → SPY adjusted close
한국: KOSPI 200 TR → KOSPI 200 Price → KODEX 200 adjusted close
```

## 7.2 초기 정책 CAGR

아래 값은 구현 초기 비교선용 임시 정책값이며, 실제 Total Return Index Rolling CAGR 검증 후 교체한다.

| 시장 | 가격수익 기준 참고값 | 총수익 기준 임시값 | 상태 |
|---|---:|---:|---|
| 미국 S&P 500 | 약 7.0% | 9.0% | 검증 필요 |
| 한국 KOSPI 200 | 약 5.8~6.0% | 6.5% | 검증 필요 |

중요:

```text
이 값은 미래 예측값이 아니다.
정적 기준 전망을 위한 정책 가정값이다.
```

정식 계산 방식:

```text
Total Return Index 월말 데이터
→ Rolling 10년 CAGR
→ 전체 Rolling 구간 중앙값
→ 정책 기준 CAGR 확정
```

## 7.3 실질수익률

실질 CAGR은 단순 차감보다 복리 조정식을 사용한다.

```text
realReturn = (1 + nominalReturn) / (1 + inflation) - 1
```

## 7.4 혼합 벤치마크

한국과 미국 자산이 혼합된 경우 CAGR을 직접 가중평균하지 않는다.

```text
Rbenchmark(t)
= wUS × RUS(t)
+ wKR × RKR(t)
```

월별 합성 수익률 경로를 만든 뒤 CAGR과 MDD를 다시 계산한다.

표시명:

```text
가중 시장 기준
미국 70% · 한국 30%
```

## 7.5 환율

원화 표시 차트에서 미국 자산은 원화 환산 수익률을 사용할 수 있다.

```text
RKRW = (1 + RUSD) × (1 + RFX) - 1
```

사용자 선택:

```text
[현지통화 기준] [원화 환산 기준]
```

기본값은 실제 원화 평가금액과 일치하도록 원화 환산 기준을 권장한다.

단, 장기 정적 CAGR 전망에서 환율을 고정적으로 예측하지 않는다. 정적 기준선과 과거 시나리오의 환율 처리 정책을 구분해 문서화한다.

## 8. 차트 및 UI 구조

## 8.1 기존 차트 유지

```text
[기준 전망] [확률 시나리오] [스트레스 테스트]
```

각 선은 동일한 투자조건을 사용한다.

```text
동일 초기 투자금
```

## 8.3 확률 시나리오 화면


시각 규칙:

- 진한 선: P50
- 옅은 밴드: P10~P90
- 점선: 고정 CAGR 기준선

## 10. 배당 처리

가격수익과 총수익을 혼합하지 않는다.

```text
Price Risk
= 일반 종가 기준 가격 하락 위험

Total Return Scenario
= 배당 재투자 조정가격 기준 투자자 총수익 경로
```

조정가격 CAGR 또는 Total Return CAGR을 사용하는 경우 배당률을 수익률에 다시 더하지 않는다.

초기 권장 정책:

```text
MDD·가격 하락 시나리오: 가격지수 또는 일반 종가
장기 총수익 시나리오: Total Return Index 또는 조정가격
```

화면에서 기준을 명확히 표시한다.

## 11. 사용자 고지

시나리오 화면 하단에 다음 고지를 표시한다.

> 시나리오 결과는 과거 수익률, 변동성 및 자산 간 움직임을 바탕으로 생성한 통계적 범위입니다. 실제 미래 수익률을 예측하거나 보장하지 않으며, 특정 금융상품의 매수·매도·보유를 권유하지 않습니다.

시장 기준선에는 다음 고지를 추가한다.

> 시장 기준 CAGR은 비교를 위한 정책 가정값 또는 과거 지수의 Rolling 수익률 통계이며, 향후 시장수익률 전망이 아닙니다.

금지 표현:

```text
향후 하락확률은 정확히 26%입니다
앞으로 MDD가 -30%를 넘지 않습니다
AI가 예측한 미래 경로입니다
시장보다 반드시 높은 수익을 냅니다
안전한 포트폴리오입니다
```

권장 표현:

```text
과거 데이터 기반 분석에서는
생성된 시나리오 중
비교 포트폴리오 대비
통계적으로 더 낮은 하락빈도를 보였습니다
분석기간과 방법에 따라 결과가 달라질 수 있습니다
```
