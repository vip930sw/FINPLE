# FINPLE Step 94. 베타 공개 후 첫 피드백 수집 운영 가이드

## 1. 단계 목적

Step 94는 FINPLE 베타 링크를 소수 사용자에게 공유한 뒤, 첫 피드백을 안정적으로 수집하고 우선순위를 판단하기 위한 운영 기준입니다.

이번 단계의 목표는 기능을 더 추가하는 것이 아니라, 실제 사용자의 반응을 기록하고 다음 개선 작업의 기준을 만드는 것입니다.

---

## 2. 베타 공개 범위

초기 공개는 소수 사용자에게만 공유합니다.

권장 범위:

- 지인 또는 내부 테스트 사용자 3~10명
- 투자 경험이 있는 사용자 2~3명
- 투자 경험이 적은 사용자 1~2명
- PC와 모바일을 모두 확인할 수 있는 사용자

초기에는 공개 범위를 크게 넓히지 않습니다. 문의사항, 서버 저장, API 조회량, 모바일 사용성에 예상하지 못한 문제가 생길 수 있기 때문입니다.

---

## 3. 공유 주소

베타 공개 주소:

```text
https://finple.co.kr
```

관리자 주소:

```text
https://finple.co.kr/admin
```

관리자 주소와 관리자 토큰은 외부 사용자에게 공유하지 않습니다.

---

## 4. 사용자에게 요청할 테스트 흐름

테스터에게는 아래 흐름으로 사용해 달라고 요청합니다.

1. FINPLE 접속
2. 홈 화면에서 서비스 설명 확인
3. 로그인 → 체험 계정으로 시작
4. 시작하기 → 포트폴리오 시뮬레이터 진입
5. 기본 포트폴리오의 자산 수량 또는 현재가 수정
6. 서버 저장 실행
7. 새로고침 후 서버 불러오기 실행
8. 상세분석 또는 포트폴리오 비교 화면 확인
9. 요금제와 투자 유의사항 확인
10. 문의사항으로 피드백 제출

---

## 5. 피드백 질문

테스터에게 아래 질문을 보내면 좋습니다.

### 첫인상

- FINPLE이 어떤 서비스인지 첫 화면에서 이해되었는가?
- 시작하기 버튼을 눌러야 한다는 흐름이 자연스러웠는가?
- 베타 안내 문구가 부담스럽거나 과하지 않았는가?

### 사용성

- 포트폴리오 입력 방식이 이해되었는가?
- 서버 저장 / 불러오기 기능을 사용할 수 있었는가?
- 시뮬레이터 화면에서 어디를 수정해야 하는지 알 수 있었는가?
- 모바일에서 불편한 화면이 있었는가?

### 투자 분석 화면

- CAGR, MDD, 배당률, 실질 평가금액 같은 용어가 이해되었는가?
- 상세분석 리포트가 읽을 만했는가?
- 포트폴리오 비교 화면이 도움이 되었는가?

### 개선 요청

- 가장 먼저 고쳤으면 하는 부분은 무엇인가?
- 추가되면 좋을 기능은 무엇인가?
- 유료 기능으로 가치가 있어 보이는 부분은 무엇인가?

## 6. 피드백 분류 기준

피드백은 아래 기준으로 분류합니다.

| 구분 | 의미 | 예시 |
|---|---|---|
| P0 | 즉시 수정 필요 | 접속 불가, 저장 실패, 관리자 조회 불가 |
| P1 | 베타 운영 중 빠른 수정 | 주요 화면 깨짐, 저장값 불일치, 문구 오해 |
| P2 | 다음 개선 후보 | UI 개선, 기능 추가, 설명 보강 |
| HOLD | 현재 보류 | 실제 결제, 정식 회원가입, Pro 기능 |

---

## 7. 관리자 문의 처리 기준

문의사항 관리자 화면에서 상태는 아래처럼 처리합니다.

| 상태 | 사용 기준 |
|---|---|
| 접수 | 새 문의가 들어온 상태 |
| 확인 중 | 내용을 검토 중인 상태 |
| 처리 완료 | 수정 또는 답변이 끝난 상태 |
| 종료 | 테스트 데이터 또는 더 이상 처리하지 않는 문의 |

테스트 문의는 운영 기록을 위해 종료 처리합니다. 삭제 기능은 추후 필요할 때 추가합니다.

---

## 8. 첫 1주일 운영 체크

베타 공개 후 첫 1주일은 아래를 매일 확인합니다.

- 사이트 접속 가능 여부
- Render 백엔드 정상 여부
- Vercel 최신 배포 상태
- 문의사항 신규 등록 여부
- 서버 저장 / 불러오기 오류 여부
- Alpha Vantage 호출 제한 관련 이슈
- 모바일 화면 주요 깨짐 여부
- 사용자가 이해하지 못한 문구 또는 용어

---

## 9. 운영 로그 작성 예시

아래 형식으로 기록합니다.

```text
날짜:
공유 대상:
신규 사용자 수:
신규 문의 수:
주요 피드백:
발견 오류:
즉시 수정 필요 여부:
다음 작업:
```

예시:

```text
날짜: 2026-05-10
공유 대상: 지인 3명
신규 사용자 수: 2명
신규 문의 수: 1건
주요 피드백: 시뮬레이터 입력칸 설명이 더 필요함
발견 오류: 없음
즉시 수정 필요 여부: 없음
다음 작업: 입력 예시 문구 검토
```

---

## 10. 공개 후 즉시 수정하지 않아도 되는 항목

아래 항목은 현재 베타 단계에서는 보류할 수 있습니다.

- 실제 이메일 회원가입
- 실제 결제 / 구독
- Pro 플랜 기능
- 모바일 시뮬레이터 전용 입력 UI
- 관리자 문의 삭제 기능
- 고급 백테스트
- 복수 사용자 권한 관리

---

## 11. 베타 공개 후 다음 단계

Step 94 완료 후 다음 단계는 아래 중 하나입니다.

1. 실제 사용자 피드백을 기반으로 P0/P1 수정
2. 입력 예시 / 도움말 문구 추가
3. 서버 저장 안정성 개선
4. 모바일 시뮬레이터 전용 UI 설계
5. 정식 회원가입 / 결제 도입 여부 검토

---

## 12. 현재 판단

Step 91 최종 실사용 테스트가 통과되었고, Step 92 베타 안내 문구와 Step 93 공유 메시지가 준비되었습니다.

따라서 FINPLE은 소수 사용자 대상 베타 공개가 가능한 상태입니다.
