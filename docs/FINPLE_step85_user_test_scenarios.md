# FINPLE Step 85. 사용자 테스트 시나리오

작성일: 2026-05-10  
목적: FINPLE 베타 공개 전, 실제 사용자가 처음 접속했을 때 주요 흐름이 자연스럽게 동작하는지 확인한다.

---

## 1. 테스트 기준

### 테스트 환경

- PC Chrome
- 모바일 Chrome 또는 Safari
- iPhone SE 크기
- Galaxy S8+ 크기
- Responsive 모드

### 테스트 주소

- 메인: https://finple.co.kr/
- 관리자: https://finple.co.kr/admin
- 개인정보처리방침: https://finple.co.kr/privacy
- 이용약관: https://finple.co.kr/terms
- 투자 유의사항: https://finple.co.kr/disclaimer

### 판정 기준

| 상태 | 의미 |
|---|---|
| PASS | 정상 동작 |
| HOLD | 운영 전 보류 가능 |
| FIX | 공개 전 수정 필요 |

---

## 2. 첫 방문자 시나리오

### 2-1. 홈 접속

1. https://finple.co.kr/ 접속
2. 상단 로고, 메뉴, 시작하기 버튼 확인
3. 홈 주요 섹션 확인
4. 하단 footer의 문서 링크 확인

판정 기준:

- 홈 화면이 깨지지 않아야 한다.
- 모바일에서 가로 스크롤이 없어야 한다.
- 투자 유의 문구 또는 문서 링크 접근이 가능해야 한다.

---

## 3. 일반 사용자 로그인 시나리오

### 3-1. 체험 계정 시작

1. 홈 화면에서 로그인 클릭
2. 일반 사용자 로그인 화면 확인
3. 관리자 로그인 영역이 보이지 않는지 확인
4. 체험 계정으로 시작 클릭
5. MY PAGE 이동 확인

판정 기준:

- 일반 로그인 화면에 관리자 카드가 없어야 한다.
- 체험 계정으로 시작 후 MY PAGE로 이동해야 한다.
- 계정 상태가 정상 표시되어야 한다.

---

## 4. 포트폴리오 시뮬레이터 시나리오

### 4-1. 기본 포트폴리오 확인

1. 시뮬레이터 이동
2. 저장된 포트폴리오 섹션 확인
3. Free 플랜에서 포트폴리오가 1개만 보이는지 확인
4. 자산 수량 변경
5. 결과값이 갱신되는지 확인

판정 기준:

- Free 플랜에서는 포트폴리오 1개만 사용 가능해야 한다.
- 수량 변경 후 평가금액, 비중, 결과값이 갱신되어야 한다.
- 모바일에서는 표가 겹치지 않고 가로 스크롤로 확인 가능해야 한다.

---

## 5. 서버 저장 / 불러오기 시나리오

### 5-1. 서버 저장

1. 포트폴리오 수량 또는 이름 변경
2. 서버 저장 클릭
3. 저장 완료 메시지 확인

### 5-2. 서버 불러오기

1. 새로고침
2. 서버 불러오기 클릭
3. 저장 시점 값으로 복구되는지 확인
4. 포트폴리오가 중복 생성되지 않는지 확인

판정 기준:

- 서버 저장이 성공해야 한다.
- 서버 불러오기 후 포트폴리오가 중복되지 않아야 한다.
- Free 플랜에서는 새로고침 후에도 포트폴리오 1개가 유지되어야 한다.

---

## 6. 문의사항 시나리오

### 6-1. 문의 작성

1. 문의사항 페이지 이동
2. 문의 유형 선택
3. 답변 이메일 입력
4. 제목과 내용 입력
5. 문의 보내기 클릭

판정 기준:

- 문의가 정상 접수되어야 한다.
- 접수 완료 메시지가 표시되어야 한다.
- 개인정보 수집 안내 문구가 보이어야 한다.

---

## 7. 관리자 시나리오

### 7-1. 관리자 로그인

1. https://finple.co.kr/admin 접속
2. 관리자 토큰 입력
3. 관리자 모드로 이동
4. MY PAGE에서 문의사항 관리자 조회 확인

판정 기준:

- /admin에서만 관리자 로그인 화면이 보여야 한다.
- 관리자 토큰 없이 일반 로그인 화면에서 관리자 영역이 보이면 안 된다.
- 관리자 토큰 입력 후 문의 목록 조회가 가능해야 한다.

### 7-2. 문의 처리

1. 문의 목록 불러오기
2. 신규 문의 확인
3. 상태를 접수 → 확인 중 → 종료로 변경
4. 전체 / 접수 / 확인 중 / 종료 필터 확인

판정 기준:

- 상태 변경이 정상 반영되어야 한다.
- 종료 처리된 문의는 종료 필터에서 확인 가능해야 한다.

---

## 8. 요금제 시나리오

### 8-1. 요금제 확인

1. 요금제 페이지 이동
2. Free / Personal / Pro 카드 확인
3. Free 선택
4. Personal 선택
5. 다시 Free 선택

판정 기준:

- 카드 스타일이 통일되어야 한다.
- 선택 상태가 브라우저에 저장되어야 한다.
- 실제 결제 기능은 준비 중임을 명확히 안내해야 한다.

---

## 9. 법적 문서 시나리오

### 9-1. 문서 링크 확인

1. footer에서 개인정보처리방침 클릭
2. 이용약관 클릭
3. 투자 유의사항 클릭
4. 각 직접 주소로도 접속

판정 기준:

- 직접 URL 접속이 가능해야 한다.
- 홈으로 돌아오면 주소가 https://finple.co.kr/ 로 정리되어야 한다.
- 모바일에서 문구가 화면 밖으로 나가면 안 된다.

---

## 10. 모바일 시나리오

### 10-1. 홈

- 상단 메뉴
- 시작하기 버튼
- 주요 카드
- 요금제 카드
- footer

### 10-2. 시뮬레이터

- 자산 스크리너
- 시뮬레이터 설정
- 포트폴리오 비교
- 상세 분석
- 저장된 포트폴리오

### 10-3. 계정/문의

- 로그인
- MY PAGE
- 문의사항
- /admin 관리자 로그인

판정 기준:

- 가로 스크롤이 불필요하게 생기지 않아야 한다.
- 표나 차트가 잘릴 경우 가로 스크롤로 확인 가능해야 한다.
- TOP 버튼이 주요 내용을 심하게 가리지 않아야 한다.
