## 6. Git 태그 생성 방법

Beta v0.1 상태를 명확히 남기려면 main 브랜치 최신 상태에서 태그를 생성합니다.

### 6.1 최신 상태 확인

```powershell
git status --short
git log --oneline -5
```

작업 중인 파일이 없어야 합니다.

### 6.2 태그 생성

```powershell
git tag -a beta-v0.1 -m "FINPLE Beta v0.1"
```

### 6.3 태그 푸시

```powershell
git push origin beta-v0.1
```

### 6.4 태그 확인

```powershell
git tag
```

또는 GitHub의 Tags / Releases 영역에서 확인합니다.

---

## 7. 태그 생성 전 주의사항

- 태그는 특정 커밋을 기준점으로 고정합니다.
- Beta v0.1 이후 수정은 새 커밋으로 진행합니다.
- 피드백 반영 후 안정화되면 `beta-v0.2` 태그를 별도로 생성합니다.
- 태그를 잘못 찍었다면 삭제 후 재생성할 수 있지만, 가능하면 최종 확인 후 생성합니다.

## 10. 운영 메모

Beta v0.1은 정식 상용 서비스가 아니라 소수 사용자 대상 검증 버전입니다.

피드백을 받을 때는 기능 요구사항보다 먼저 다음을 확인합니다.

1. 사용자가 서비스 목적을 바로 이해하는가
2. 시뮬레이터 입력이 어렵지 않은가
3. 결과 차트와 리포트를 이해할 수 있는가
4. 저장/불러오기를 자연스럽게 사용할 수 있는가
5. 문의사항으로 피드백을 남길 수 있는가

이 기준을 통과하면 다음 단계는 Beta v0.2 피드백 반영입니다.
