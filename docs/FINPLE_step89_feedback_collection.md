# FINPLE Step 89. 사용자 피드백 수집 방식 정리

## 1. 목적

이 문서는 FINPLE 베타 공개 이후 사용자 피드백을 어떤 방식으로 수집하고, 어떤 기준으로 우선순위를 나눌지 정리하기 위한 운영 기준입니다.

베타 단계에서는 기능을 많이 늘리는 것보다, 실제 사용자가 어디에서 막히는지, 어떤 기능을 가장 먼저 필요로 하는지, 투자 분석 결과를 이해하기 쉬운지 확인하는 것이 중요합니다.

---

## 2. 피드백 수집 채널

### 2.1 사이트 내 문의사항

현재 FINPLE의 기본 피드백 채널입니다.

- 접속 경로: FINPLE → 문의사항
- 저장 위치: 서버 inquiries 데이터
- 관리자 확인: /admin → 관리자 토큰 입력 → MY PAGE 관리자 문의 조회

권장 사용 목적:

- 오류 신고
- 기능 제안
- 데이터 문의
- 화면 사용성 문제
- 결제/요금제 관련 의견

### 2.2 직접 메시지

초기 베타 사용자에게는 카카오톡, 문자, 이메일 등 직접 메시지를 병행할 수 있습니다.

단, 운영 기록을 남기기 위해 중요한 피드백은 관리자가 직접 문의사항에 요약 등록하거나 Step 90 운영 로그에 옮겨 적습니다.

### 2.3 구글폼 또는 설문

베타 사용자 수가 늘어나면 별도 설문을 사용할 수 있습니다.

추천 질문:

1. FINPLE을 처음 봤을 때 어떤 서비스인지 이해되었나요?
2. 가장 유용했던 기능은 무엇인가요?
3. 가장 헷갈렸던 화면은 어디인가요?
4. 포트폴리오 저장/불러오기는 자연스러웠나요?
5. 투자 유의 문구와 분석 결과가 충분히 명확했나요?
6. 모바일에서 불편한 부분이 있었나요?
7. 유료 기능으로 필요하다고 느낀 기능이 있었나요?

## 3. 피드백 분류 기준

피드백은 아래 5개 유형으로 분류합니다.

| 구분 | 의미 | 예시 | 처리 우선순위 |
|---|---|---|---|
| 오류 | 기능이 정상 작동하지 않음 | 저장 실패, 화면 깨짐, 차트 오류 | 최우선 |
| 사용성 | 작동은 하지만 불편함 | 버튼 위치 혼란, 문구 이해 어려움 | 높음 |
| 데이터 | 가격/지표/티커 관련 문제 | 현재가 오류, 종목 검색 실패 | 높음 |
| 기능 제안 | 새로운 기능 요청 | PDF 고도화, 리밸런싱 알림 | 중간 |
| 정책/문구 | 약관·투자 유의·요금제 관련 의견 | 유의 문구 부족, 가격 혼란 | 중간 |

## 4. 긴급도 판단 기준

### P0. 즉시 대응

서비스 신뢰에 직접 영향을 주는 문제입니다.

- 사이트 접속 불가
- 로그인/체험 계정 진입 불가
- 서버 저장/불러오기 실패
- 문의사항 등록 불가
- 관리자 문의 조회 불가
- 개인정보 또는 토큰 노출 가능성
- 투자 추천처럼 오해될 수 있는 문구

### P1. 빠른 수정

베타 공개 중 빠르게 개선하면 좋은 문제입니다.

- 모바일 주요 화면 깨짐
- 요금제 문구 혼란
- 저장 후 새로고침 시 값 불일치
- 포트폴리오 개수 제한 오류
- 차트 일부 잘림
- 주요 버튼 위치 혼란

### P2. 다음 개선

운영 중 backlog로 관리할 수 있는 문제입니다.

- 모바일 시뮬레이터 고도화
- 실제 이메일 로그인
- 결제/구독 연동
- 리밸런싱 자동 제안
- 고급 백테스트
- 사용자별 히스토리 관리

---

## 5. 피드백 처리 흐름

1. 사용자가 문의사항 또는 직접 메시지로 피드백 제출
2. 관리자가 /admin에서 문의 확인
3. 상태를 접수 → 확인 중 → 종료로 변경
4. 중요한 내용은 Step 90 운영 로그에 기록
5. 오류 또는 기능 개선이 필요한 경우 backlog로 분류
6. 수정 후 베타 사용자에게 반영 여부 안내

---

## 6. 관리자 문의 상태 기준

| 상태 | 의미 | 사용 기준 |
|---|---|---|
| 접수 | 새로 들어온 문의 | 아직 확인하지 않은 상태 |
| 확인 중 | 검토 또는 수정 진행 | 원인 확인, 패치 준비, 재현 테스트 중 |
| 처리 완료 | 수정 또는 답변 완료 | 사용자가 확인하면 종료 가능 |
| 종료 | 더 이상 조치 필요 없음 | 테스트 문의, 단순 문의, 완료된 문의 |

---

## 7. 베타 사용자에게 요청할 피드백 문구

### 짧은 요청 문구

FINPLE은 현재 베타 테스트 중입니다. 사용 중 불편한 점, 이해하기 어려운 문구, 오류가 보이면 문의사항 메뉴로 알려주세요. 실제 투자 판단 전에 반드시 별도 검토가 필요합니다.

### 지인 공유용 문구

FINPLE 베타 버전을 공개했습니다. 포트폴리오를 입력하면 장기 성과와 위험 지표를 확인할 수 있는 투자 분석 도구입니다. 아직 체험 운영 단계라 실제 로그인과 결제는 연결되어 있지 않습니다. 사용해보시고 불편한 부분이나 필요한 기능을 알려주시면 개선에 반영하겠습니다.

---

## 8. 피드백 기록 양식

피드백을 운영 로그에 옮길 때는 아래 형식을 사용합니다.

| 항목 | 내용 |
|---|---|
| 접수일 | YYYY-MM-DD |
| 제보자 | 사용자명 또는 익명 |
| 화면 | 홈 / 로그인 / 시뮬레이터 / 문의사항 / 관리자 / 요금제 |
| 유형 | 오류 / 사용성 / 데이터 / 기능 제안 / 정책 문구 |
| 내용 | 사용자가 전달한 내용 요약 |
| 긴급도 | P0 / P1 / P2 |
| 처리 상태 | 접수 / 확인 중 / 처리 완료 / 종료 |
| 조치 내용 | 수정 사항 또는 보류 사유 |
| 후속 작업 | 필요 시 다음 Step 또는 backlog 연결 |

---

## 9. 베타 기간 중 확인할 핵심 질문

베타 테스트에서 가장 중요한 질문은 아래입니다.

1. 사용자가 FINPLE의 목적을 바로 이해하는가?
2. 체험 계정 로그인 흐름이 자연스러운가?
3. 포트폴리오 1개 제한이 혼란스럽지 않은가?
4. 서버 저장/불러오기 개념을 이해하는가?
5. 분석 결과가 투자 추천처럼 오해되지 않는가?
6. 모바일에서 최소한 핵심 화면을 볼 수 있는가?
7. 유료화한다면 어떤 기능부터 돈을 낼 가치가 있는가?
