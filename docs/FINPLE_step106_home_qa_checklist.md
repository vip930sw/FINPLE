# FINPLE Step 106. 홈 화면 최종 QA 체크리스트

## 1. 목적

Step 106은 기능 추가 단계가 아니라, 베타 공개 전 홈 화면을 실제 사용자 관점에서 최종 점검하는 단계입니다.

주요 목적은 다음과 같습니다.

- 첫 방문자가 FINPLE의 목적을 빠르게 이해하는지 확인
- Why / How / What 흐름이 자연스러운지 확인
- 시뮬레이터 진입 동선이 충분히 명확한지 확인
- 모바일 화면에서 홈 페이지가 과도하게 길거나 무겁지 않은지 확인
- 베타 운영 중 불필요한 개발자용 표현이 노출되지 않는지 확인
- 다음 단계인 임시 파일 정리와 베타 v0.1 마감 작업 전에 화면 품질을 고정

---

## 2. 확인 환경

아래 환경에서 최소 1회씩 확인합니다.

| 구분 | 확인 기준 |
|---|---|
| PC Chrome | https://finple.co.kr 직접 접속 |
| PC 시크릿 모드 | 기존 localStorage 영향 없이 첫 방문 확인 |
| 모바일 에뮬레이션 | iPhone SE / Galaxy S8+ / Responsive |
| 실제 모바일 | 가능하면 카카오톡 링크 진입 기준 확인 |
| 직접 URL | /simulator, /pricing, /support, /login, /privacy, /terms, /disclaimer |

## 3. 홈 화면 전체 흐름 QA

### 3-1. Hero

확인 항목:

- [ ] 로고 / 내비바 / 로그인 / 시작하기 / 문의사항 버튼이 정상 표시된다.
- [ ] "내 포트폴리오의 수익과 위험을 한눈에 분석하세요" 문구가 첫 화면에서 자연스럽다.
- [ ] Beta 안내 문구가 과하지 않고, 투자 유의 문구로서 적절하다.
- [ ] "무료로 시작하기" 버튼 클릭 시 /simulator로 이동한다.
- [ ] "기능 살펴보기" 버튼 클릭 시 Features 또는 How 영역으로 이동한다.
- [ ] 오른쪽 예상 포트폴리오 카드의 항목명이 성장주 / 가치주 / 채권 / 금으로 표시된다.

### 3-2. Economic Calendar

확인 항목:

- [ ] Problem 섹션보다 위에 위치한다.
- [ ] 경제 일정 위젯이 정상 표시된다.
- [ ] PC에서 너무 크거나 깨지지 않는다.
- [ ] 모바일에서 화면 밖으로 튀어나가지 않는다.
- [ ] 로딩 지연 시에도 홈 전체 흐름을 크게 방해하지 않는다.

### 3-3. Why

확인 항목:

- [ ] "왜 이 포트폴리오 시뮬레이터를 사용해야 하는지"가 전달된다.
- [ ] 단기 수익률보다 장기 구조, 위험 대응, 자산배분 관점이 강조된다.
- [ ] 출처 카드가 자연스럽게 보인다.
- [ ] 출처 링크 클릭이 정상 작동한다.
- [ ] 문구가 과장된 수익 보장처럼 보이지 않는다.
- [ ] 분위기 이미지형 패널이 서비스 성격과 어울린다.

### 3-4. How

확인 항목:

- [ ] 제목이 "포트폴리오 시뮬레이터를 통해 성장과 위험에 대응합니다."로 표시된다.
- [ ] FINPLE이 단순 입력표가 아니라 의사결정 흐름을 제공한다는 점이 전달된다.
- [ ] 카드 문구가 "..."으로 잘리지 않는다.
- [ ] 카드 6개가 PC에서 균형 있게 보인다.
- [ ] 모바일에서 카드가 세로로 자연스럽게 쌓인다.
- [ ] 섹션 폭이 과도하게 좁거나 넓어 보이지 않는다.

### 3-5. What

확인 항목:

- [ ] 제목이 "FINPLE은 다양한 투자 의사결정과 대안을 제시합니다."로 표시된다.
- [ ] 사용자가 얻는 결과물, 경험, 만족도가 전달된다.
- [ ] "자산흐름을 시각화" 문구가 정상 표시된다.
- [ ] 일러스트형 이미지 카드가 자연스럽다.
- [ ] 카드 문구가 "..."으로 잘리지 않는다.
- [ ] 투자 MBTI 테스트 Coming Soon 카드가 과하게 튀지 않는다.
- [ ] 실제 기능으로 오해되지 않도록 Coming Soon 상태가 명확하다.

### 3-6. Demo Calculator

확인 항목:

- [ ] Pricing 바로 위에 위치한다.
- [ ] 간단한 계산기라는 목적이 명확하다.
- [ ] PC에서 입력창이 지나치게 크지 않다.
- [ ] 모바일에서 과도하게 길어 보이지 않는다.

### 3-7. Pricing

확인 항목:

- [ ] Free / Personal / Pro 카드 문구가 자연스럽다.
- [ ] Personal 카드만 CSS가 어긋나 보이지 않는다.
- [ ] 하단 투자 유의 문구가 보인다.
- [ ] 카드 하단 여백이 충분하다.
- [ ] 결제 기능이 아직 준비 중이라는 점이 명확하다.

### 3-8. Footer

확인 항목:

- [ ] 개인정보처리방침 / 이용약관 / 투자 유의사항 링크가 보인다.
- [ ] 각 문서 페이지로 이동한다.
- [ ] 홈으로 돌아왔을 때 URL이 / 로 정리된다.
- [ ] 투자 유의 문구가 과하지 않지만 명확하다.

## 5. 시뮬레이터 진입 QA

확인 항목:

- [ ] /simulator 직접 접속이 가능하다.
- [ ] 새로고침 후에도 /simulator가 유지된다.
- [ ] 투자기간 기본값이 10년으로 표시된다.
- [ ] 새로고침 후에도 10년이 유지된다.
- [ ] 기존 30년 localStorage 값이 기본값인 경우 10년으로 보정된다.
- [ ] 사용자가 직접 투자기간을 변경하면 변경값이 유지된다.

## 6. 복사 방지 QA

확인 항목:

- [ ] 일반 본문 영역에서 오른쪽 클릭이 막힌다.
- [ ] 일반 본문 영역에서 드래그 선택이 막힌다.
- [ ] 일반 본문 영역에서 복사가 막힌다.
- [ ] 입력창에서는 텍스트 입력과 수정이 정상 작동한다.
- [ ] 버튼과 링크는 정상 클릭된다.
- [ ] 모바일 터치 조작에 문제가 없다.

주의:
복사 방지는 완전한 보안 기능이 아닙니다. 개발자도구, 스크린샷, 외부 캡처까지 막을 수는 없습니다.
