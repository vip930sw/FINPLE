# FINPLE Step 103-15

## 목적

CSV 후보군을 한/미 각각 2,000개 수준까지 확장할 수 있도록 스크리너 화면에 페이지 단위 표시 구조를 먼저 적용합니다.

후보군이 1,000개 이상으로 늘어나면 필터를 걸지 않은 상태에서 전체 카드가 한 번에 출력될 수 있으므로, 화면 과부하와 스크롤 피로도를 줄이기 위해 20 / 50 / 100개 단위 표시와 페이지 이동을 적용합니다.

## 이번 PR 범위

```text
1. 스크리너 결과 페이지네이션 추가
2. 표시 개수 선택 추가
   - 20개
   - 50개
   - 100개
3. 필터, 검색어, 시장, 자산군, 투자 스타일, 위험도, 표시 개수 변경 시 1페이지로 자동 이동
4. 현재 표시 범위 안내
   - 예: 1,000개 후보 중 1-20 표시
5. 처음 / 이전 / 페이지 번호 / 다음 / 끝 버튼 추가
6. 모바일에서도 페이지네이션이 깨지지 않도록 반응형 스타일 적용
```

## 후보군 확장 방향

최종 목표는 아래 수준으로 둡니다.

```text
미국 후보: 약 2,000개
한국 후보: 약 2,000개
총 후보: 약 4,000개
```

단, 실제 앱 반영은 한 번에 4,000개를 넣지 않고 단계적으로 진행합니다.

```text
1차: 미국 500개 + 한국 500개 = 약 1,000개
2차: 미국 1,000개 + 한국 1,000개 = 약 2,000개
3차: 미국 2,000개 + 한국 2,000개 = 약 4,000개
```

## CSV 기본 컬럼

현재 앱 로더와 호환되는 기본 컬럼은 아래와 같습니다.


## 후보군 확장 시 추가 권장 컬럼

대규모 후보군 관리에는 아래 컬럼 추가를 권장합니다.

```text
tier
sector
industry
country
dataStatus
liquidityRank
sourceUniverse
```

`tier`는 후보 노출 우선순위를 관리하기 위한 컬럼입니다.

```text
core      : 기본 노출 우선 후보
standard  : 일반 검색 후보
extended  : 검색어 또는 필터 일치 시 노출 후보
```

## Colab 가공 기준

후보군 CSV는 Colab에서 다음 순서로 가공합니다.

```text
1. ticker / market 기준 중복 제거
2. 미국 ticker는 BRK.B -> BRK-B 등 providerSymbol 정규화
3. 한국 ticker는 6자리 문자열 유지
4. ETF / stock assetType 분리
5. strategy / riskLevel / goals / tags 자동 분류
6. v2.2.3 metrics override와 매칭
7. 지표 없는 후보는 후보로 유지하되 dataStatus = pending_metrics 처리
8. 앱 반영 전 공란 비율 진단
```

## 화면 검수 체크리스트

```text
1. 필터 미적용 시 후보가 20개만 표시되는지
2. 표시 개수를 50 / 100개로 변경할 수 있는지
3. 다음 / 이전 / 처음 / 끝 버튼이 작동하는지
4. 검색어 입력 시 1페이지로 돌아가는지
5. 시장 / 자산군 / 투자 스타일 / 위험도 변경 시 1페이지로 돌아가는지
6. 모바일에서 페이지네이션 버튼이 깨지지 않는지
7. 대량 CSV 반영 후 화면 로딩이 과도하게 느려지지 않는지
```
