# FINPLE Step 109. Beta v0.1 공개 후 24시간 모니터링 체크리스트

## 1. 목적

Step 109는 FINPLE Beta v0.1을 소수 사용자에게 공유한 뒤, 첫 24시간 동안 반드시 확인해야 할 운영 항목을 정리하는 문서입니다.

이 단계의 목표는 새 기능을 추가하는 것이 아니라, 실제 사용 중 발생할 수 있는 접속 오류, 저장 오류, 문의 누락, 모바일 사용 불편을 빠르게 발견하고 우선순위를 나누는 것입니다.

---

## 2. 24시간 모니터링 원칙

베타 공개 직후에는 기능을 즉시 늘리지 않습니다.

우선 아래 항목을 관찰합니다.

- 사용자가 정상 접속하는지
- 체험 계정으로 진입하는지
- 포트폴리오 입력에서 막히는지
- 서버 저장 / 불러오기가 정상 작동하는지
- 문의사항이 정상 접수되는지
- 관리자 화면에서 문의를 확인할 수 있는지
- 모바일에서 주요 화면이 깨지지 않는지
- 투자 유의 문구가 과하거나 부족하지 않은지

## 3. 공개 직후 1시간 체크

### 3-1. 접속 상태

- [ ] https://finple.co.kr 접속 가능
- [ ] 홈 화면 정상 표시
- [ ] 브라우저 탭 제목 정상 표시
- [ ] OG 미리보기 이미지 정상 표시 또는 캐시 반영 대기 확인

### 3-2. 핵심 링크

- [ ] /simulator 직접 접속 가능
- [ ] /pricing 직접 접속 가능
- [ ] /support 직접 접속 가능
- [ ] /privacy 직접 접속 가능
- [ ] /terms 직접 접속 가능
- [ ] /disclaimer 직접 접속 가능
- [ ] /admin 직접 접속 가능

### 3-3. 백엔드 상태

- [ ] Render 백엔드 Health Check 정상
- [ ] 첫 요청 지연이 있어도 30~60초 내 정상 복귀
- [ ] 서버 저장 정상
- [ ] 서버 불러오기 정상

## 4. 공개 후 6시간 체크

### 4-1. 사용자 문의

- [ ] 문의사항 신규 접수 여부 확인
- [ ] 관리자 화면에서 문의 조회 가능
- [ ] 상태 변경 가능
- [ ] 테스트 문의와 실제 문의 구분 가능

### 4-2. 사용자 사용 흐름

사용자에게 아래 흐름 중 어디서 막혔는지 확인합니다.

- 홈 이해
- 체험 계정 시작
- 시뮬레이터 진입
- 포트폴리오 입력
- 자산 조회
- 결과 차트 이해
- 서버 저장 / 불러오기
- 문의사항 작성

### 4-3. 모바일 사용성

- [ ] 모바일 홈 화면 확인
- [ ] 모바일 로그인 화면 확인
- [ ] 모바일 문의사항 화면 확인
- [ ] 모바일 시뮬레이터 화면 확인
- [ ] 가로 스크롤이 필요한 표에서 스크롤 가능 여부 확인

---

## 5. 공개 후 24시간 체크

### 5-1. 데이터 누락 / 오류

- [ ] Supabase 문의 데이터 정상 저장
- [ ] 포트폴리오 데이터 정상 저장
- [ ] 서버 저장 중복 발생 여부 확인
- [ ] Free 플랜 1개 포트폴리오 제한 유지
- [ ] 투자기간 기본값 10년 유지

### 5-2. 오류 분류

| 분류 | 기준 | 대응 |
|---|---|---|
| P0 | 접속 불가, 저장 불가, 문의 불가 | 즉시 수정 |
| P1 | 사용은 가능하지만 핵심 흐름에 불편 | 우선 수정 후보 |
| P2 | 문구, 디자인, 모바일 가독성 | 피드백 누적 후 수정 |
| HOLD | 결제, 실제 로그인, Pro 기능 | Beta v0.2 이후 검토 |

### 5-3. 피드백 기록

피드백은 아래 형식으로 기록합니다.

```text
일시:
사용자 유형:
사용 기기:
사용 화면:
발생 문제:
사용자 코멘트:
분류: P0 / P1 / P2 / HOLD
다음 조치:
```

## 6. 첫 24시간에 바로 고치지 않아도 되는 항목

아래 항목은 치명적인 오류가 아니라면 즉시 수정하지 않고 누적 관찰합니다.

- 색상 선호도
- 카드 간격
- 일부 문구 취향
- 투자 MBTI 테스트 요청
- 유료 플랜 상세 기능 요청
- 실제 회원가입 / 결제 요청
- Pro 기능 요청

## 7. 첫 24시간에 바로 고쳐야 하는 항목

아래 항목은 즉시 대응합니다.

- 사이트 접속 불가
- /simulator 직접 접속 불가
- 체험 계정 시작 불가
- 포트폴리오 저장 불가
- 서버 불러오기 불가
- 문의사항 접수 불가
- 관리자 조회 불가
- 모바일에서 핵심 버튼 클릭 불가
- 법적 문서 접근 불가

## 8. 사용자에게 추가로 물어볼 질문

베타 사용 후 아래 질문을 짧게 물어봅니다.

1. 처음 화면에서 FINPLE이 어떤 서비스인지 바로 이해됐나요?
2. 시뮬레이터 입력은 어렵지 않았나요?
3. 결과 차트와 수치가 이해됐나요?
4. 저장 / 불러오기 흐름이 자연스러웠나요?
5. 투자 판단에 실제로 도움이 될 것 같았나요?
6. 가장 먼저 개선했으면 하는 부분은 무엇인가요?
