## 9. 배포 / 환경변수 점검

| 항목 | 확인 결과 | 비고 |
|---|---:|---|
| Vercel Production 배포 | 완료 | 프론트 |
| Render Live 상태 | 완료 | 백엔드 |
| CORS_ORIGIN | 완료 | finple.co.kr, www 포함 |
| FINPLE_ADMIN_TOKEN | 완료 | Render 환경변수 |
| DATABASE_URL | 완료 | Render 환경변수 |
| DATABASE_SSL | 완료 | true |
| Alpha Vantage API Key | 완료 | Render 환경변수 |
| VITE_FINPLE_API_BASE_URL | 완료 | 프론트 API 주소 |

운영 기준:

- API Key, DATABASE_URL, ADMIN_TOKEN은 Vercel 프론트 환경변수에 넣지 않습니다.
- 브라우저에 노출되어도 되는 값만 VITE_ 변수로 사용합니다.
- 백엔드 비밀값은 Render 환경변수에서 관리합니다.

## 11. 공개 전 최종 테스트 순서

운영 공개 직전 아래 순서로 다시 한 번 확인합니다.

1. https://finple.co.kr/ 접속
2. 홈 화면 로딩 확인
3. 로그인 → 개발 사용자 로그인
4. MY PAGE 진입
5. 포트폴리오 수량 변경
6. 서버 저장
7. 새로고침
8. 서버 불러오기
9. 값 복구 확인
10. 문의사항 작성
11. /admin 접속
12. 관리자 토큰 입력
13. 문의 목록 조회
14. 신규 문의 상태 변경
15. 관리자 모드 해제
16. 개인정보처리방침 / 이용약관 / 투자 유의사항 링크 확인
17. 모바일 화면 핵심 메뉴 확인

## 12. 현재 결론

현재 FINPLE은 초기 공개 테스트가 가능한 수준까지 정리되었습니다.

다만 실제 유료 서비스로 전환하기 전에는 다음 작업이 필요합니다.

1. 실제 회원 인증 구조

4. 개인정보처리방침/이용약관 법무 검토
5. 관리자 권한 고도화
6. 운영 로그와 장애 대응 기준 정리
